PROTOCOLO HTTP

HyperText Transfer Protocol

O Brower implementa o protocolo HTTP

Clientes: mensagens - Request HTTP

Servidor: mensagens - Response HTTP

GET, POST

GET - solicita dados do servidor

POST - envia dados ao servidor

XML E JSON

XML - metadados mais complexos

JSON - metadados mais simples

O Browser permite até 6 requests antes de esperar a resposta do servidor

Resquest e Respose

Request

Dividida em Request Line e Header Line

Get:/algumDiretorio/page.html HTTP/1.1

Host: www.algumhost.edu

Connection: close (não persistente)

User agent: Mozilla/5.0

Accept Lang: pt-br (lingua preferencial)

GET - Solicita um rescurso do Servidor

HEAD - GET sem corpo de resposta

POST - Submete uma entidade a um recurso

PUT - Substituição de recursos por dados da requisição

DELETE - remoção de um recurso

TRACE - chamada de loop-back a um determinado recurso

OPTION - opções de comunicacação com recurso

CONNECT - tunelamentoidentificado pelo recurso

PATCH - modificação parcial

Metodos Seguros - GET, HEAD e OPTION

Response

Dividida em Status Line, Header Line e entity body

Status Code

(100-199) informação

(200-299) Sucesso

(300-399) Redirecionamento

(400 -499) Erro no Cliente

(500-599) Erro no Servidor

Cookies e Chache

Cookies

Pequenos pedaços ou blocos de dados criados e utilizados pelo servidor para persistir dados no dispositivo do cliente.

Ou seja: rastrear o cliente quando estiver no site

Pode ser Persistente (ex: Longin ativo, n requisita dados novamente) ou de sessão (Exemplo: encerra junto com a sessão, requer dados delogin novamente)

Cache

Armazena dados do servidor em um Proxy, servido como um Proxy server se os dados solicitados pelo clinte estiverem nele.

Caso não estejam, ele solicita ao servidor, armazena os dados e envia ao cliente

Vantagens: Redução de tempo de resposta e de tráfedo na rede
